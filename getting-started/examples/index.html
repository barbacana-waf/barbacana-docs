    Configuration examples¶

    Short, task-focused configurations for common deployments. Pick the one closest to your setup and adapt. Each example links to the reference pages that explain every field it uses.

    Single app behind a load balancer¶

    Your LB terminates TLS and forwards plain HTTP. Barbacana just inspects and proxies.

    waf.yaml
    version: v1alpha1
port: 8080

routes:
  - upstream: http://app:8000

    See Hostnames & HTTPS → Mode 3.

    Single app with auto-HTTPS¶

    Barbacana provisions a Let's Encrypt certificate for one hostname. Mount data_dir as a persistent volume so certs survive restarts.

    waf.yaml
    version: v1alpha1
host: api.example.com
data_dir: /data/barbacana

routes:
  - upstream: http://app:8000

    See Hostnames & HTTPS → Mode 1.

    Multi-host deployment¶

    Several public hostnames through one Barbacana; one certificate per host.

    waf.yaml
    version: v1alpha1
data_dir: /data/barbacana

routes:
  - id: api
    match: { hosts: [api.example.com] }
    upstream: http://api:8000

  - id: web
    match: { hosts: [www.example.com] }
    upstream: http://web:8000

    See Hostnames & HTTPS → Mode 2.

    Disable a protection on one route¶

    A legitimate search endpoint trips sql-injection-union. Turn off only that technique, only for that route.

    routes:
  - id: search
    match: { paths: ["/search"] }
    upstream: http://search:8000
    disable:
      - sql-injection-union

    See Disable protections and the full catalog.

    Onboard in detect-only, then switch to blocking¶

    Log without blocking while you tune false positives, then flip once clean.

    routes:
  - id: legacy
    match: { paths: ["/legacy/*"] }
    upstream: http://legacy:8000
    mode: detect_only       # later: remove this line (default is "blocking")

    See Detect-only mode.

    JSON-only API on a path prefix¶

    Only JSON, only declared methods, with the public /v1 prefix stripped before forwarding.

    routes:
  - id: public-api
    match: { paths: ["/v1/*"] }
    upstream: http://api:8000
    accept:
      content_types: [application/json]
      methods: [GET, POST, PUT, DELETE]
    rewrite:
      strip_prefix: /v1

    See Accept and Rewrites.

    Validate requests against an OpenAPI spec¶

    Enforce your contract — path, method, params, body — at the edge.

    routes:
  - id: public-api
    match: { paths: ["/v1/*"] }
    upstream: http://api:8000
    openapi:
      spec: /etc/barbacana/specs/public-api.yaml
      strict: true

    See OpenAPI.

    File upload route¶

    Multipart only, capped file count and size, restricted MIME types, double-extension rejection.

    routes:
  - id: uploads
    match: { paths: ["/upload/*"] }
    upstream: http://uploads:8000
    accept:
      content_types: [multipart/form-data]
      max_body_size: 50MB
    multipart:
      file_limit: 10
      file_size: 5MB
      allowed_types: [image/png, image/jpeg, application/pdf]
      double_extension: true

    See Accept and Uploads.

    Public API with CORS¶

    Allow a known SPA origin, credentials, and a short preflight cache.

    routes:
  - id: api
    match: { paths: ["/api/*"] }
    upstream: http://api:8000
    cors:
      allow_origins: ["https://app.example.com"]
      allow_methods: [GET, POST]
      allow_headers: [Authorization, Content-Type]
      allow_credentials: true
      max_age: 600

    See CORS.

    Tighter security headers for the admin panel¶

    Override the default moderate preset with strict and a custom CSP.

    routes:
  - id: admin
    match: { paths: ["/admin/*"] }
    upstream: http://admin:8000
    response_headers:
      preset: strict
      inject:
        header-csp: "default-src 'self'; frame-ancestors 'none'"

    See Security headers.

    Mixed public + admin + catch-all¶

    A common full layout: one API route with a spec, one admin route with strict headers, one catch-all for everything else.

    waf.yaml
    version: v1alpha1
host: example.com
data_dir: /data/barbacana

routes:
  - id: api
    match: { paths: ["/api/*"] }
    upstream: http://api:8000
    accept:
      content_types: [application/json]
    rewrite: { strip_prefix: /api }
    openapi: { spec: /specs/api.yaml }

  - id: admin
    match: { paths: ["/admin/*"] }
    upstream: http://admin:8000
    response_headers: { preset: strict }

  - id: everything-else
    upstream: http://app:8000
